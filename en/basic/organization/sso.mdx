title: "Single Sign-On (SSO)"
description: "Integrate Single Sign-On (SSO) with support for standard OIDC and Feishu/Lark"
---

<Tip>Available for Enterprise Edition</Tip>

Supported SSO Providers:

* Standard OIDC
* Feishu/Lark

## Feishu/Lark Integration

### 1. Create a Custom App in Feishu/Lark

**Feishu Client -> Workplace -> Create App**

Obtain the App ID and App Secret

![](/images/kai-fa-zhe/ji-cheng/dan-dian-deng-lu/eafb64024965053a.png)

### 2. Create Authentication Provider in Teable

**Organization Management Panel -> Authentication Settings**

<Note>Requires organization administrator account</Note>

![](/images/kai-fa-zhe/ji-cheng/dan-dian-deng-lu/d9e424732c7bd9c7.png)

#### Enter APP ID and APP Secret

These values are obtained when creating the custom app in step 1

![](/images/kai-fa-zhe/ji-cheng/dan-dian-deng-lu/8f8c41d77bde45a2.png)

**Click to copy the providerId for later use**

![](/images/kai-fa-zhe/ji-cheng/dan-dian-deng-lu/49353a708fc1f1e1.png)

### 3. Return to Feishu/Lark App Configuration

#### 3.1 Permission Settings

Development Configuration -> Permission Management

Search and select:

`contact:user.base:readonly`

`contact:user.email:readonly`

#### 3.2 Security Settings

Development Configuration -> Security Settings

Redirect URL:

`https://app.teable.io/api/auth/authentication/__providerId__/callback`

Replace `__providerId__` with the providerId copied in step 2

For self-hosted instances, replace `app.teable.io` with your instance's domain or IP

#### 3.3 Web Application Settings

App Capabilities -> Web App -> Web App Configuration, click Settings -> Desktop Homepage, Mobile Homepage:

Same as the redirect URL:

`https://app.teable.io/api/auth/authentication/__providerId__`

### 4. Publish Version

Click Create Version -> Upload Icon

![](/images/kai-fa-zhe/ji-cheng/dan-dian-deng-lu/66f7441d660f2858.png)

Set version number (e.g., 1.0.0)
Choose availability scope (e.g., visible to all)
Click publish and complete the approval process

### 5. Access the Application
<Tabs>
<Tab title="Cloud Version">

Cloud login URL requires your `organizationId`
For example: `https://app.teable.io/auth/login?organizationId=orgxxxxxxxx`
The login page will then show the added Feishu/Lark login button, or you can click the app icon directly in Feishu/Lark

</Tab>
<Tab title="Self-Hosted">

For self-hosted version:
Simply access the application URL to see the login button, or click the app icon in Feishu/Lark
![](/images/kai-fa-zhe/ji-cheng/dan-dian-deng-lu/54501614ff7424f1.png)
</Tab>
</Tabs>
